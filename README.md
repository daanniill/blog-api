# blog-api

A REST API for managing my blog posts made to daanniill.github.io, built with Express and PostgreSQL (Supabase).

## Tech stack

- Node.js / Express 5
- PostgreSQL via `pg`, Supabase
- dotenv + dotenv-expand for config

## Setup

```bash
npm install
```

Create a `.env` file in the project root:

```
db_pass=your_db_password
DATABASE_URL=postgresql://postgres:${db_pass}@your-host:5432/postgres
ADMIN_API_KEY=your_admin_key
PORT=3000
```

`ADMIN_API_KEY` is required as the `x-api-key` header on write endpoints (create/update/delete).

## Running

```bash
npm run dev    # nodemon, auto-restart
npm start      # plain node
```

## API Endpoints

### Health

- GET: `/api/health`; Check database connection

### Posts

- GET: `/api/posts`; Get all posts
- GET: `/api/posts/:id`; Get a post by ID
- GET: `/api/posts/slug/:slug`; Get a post by slug
- POST: `/api/posts`; Create a post
- PATCH: `/api/posts/:id`; Update a post
- DELETE: `/api/posts/:id`; Delete a post

> Creating, updating, and deleting posts requires authentication.

Auth routes require an `x-api-key` header matching `ADMIN_API_KEY`.

**Post fields:** `title`, `slug`, `summary`, `content`, `cover_image_url` (`id` is auto-generated by the database).
